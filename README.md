# 🚀 Pyrogram Post Formatter (User Client)

**Асинхронний Post Formatter, що працює від вашого особистого Telegram-акаунту**

Це приватна версія бота, яка працює через **User API** (замість Bot API), дозволяючи операціям здійснюватися від вашого особистого акаунту. Ви можете надсилати повідомлення приватно клієнту, і він автоматично форматуватиме текст та публікуватиме в ваш канал.

## ✨ Основні функції

- ✅ **Автоматична заміна ключових слів на HTML-посилання**
  - Зберігає оригінальний регістр слова
  - Пошук без урахування регістру
  - Легко редагується словник

- ✅ **Публікація текстових постів**
  - Надішліть текст приватно клієнту
  - Клієнт автоматично замініть ключові слова та опублікує у ваш канал

- ✅ **Публікація фото з форматованим описом**
  - Обробка опису за тією ж логікою
  - Збереження якості фото

- ✅ **Асинхронна обробка** (Pyrogram)
  - Швидка відповідь на повідомлення
  - Оптимальне використання ресурсів

- ✅ **Комунікація через приватні повідомлення**
  - Надсилайте повідомлення приватно
  - Отримуйте підтвердження про публікацію

## 🔧 Відмінності від Bot API версії

| Параметр | Bot API | User API |
|----------|---------|----------|
| **Реєстрація** | Через @BotFather | Через особистий акаунт |
| **Доступ** | Надсилаєте команди боту | Надсилаєте приватні повідомлення |
| **Автентифікація** | Token | API ID + API Hash |
| **Рейт-ліміти** | Більш м'яких | Білі м'яки |
| **Типи фільтрів** | Деякі обмеження | Більш гнучкий доступ |

## 🚀 Встановлення та запуск

### Вимоги
- Python 3.8+
- pip (менеджер пакетів Python)
- Особистий Telegram-акаунт

### Крок 1: Отримання необхідних даних

#### 1.1 API ID та API Hash
1. Перейдіть на https://my.telegram.org/apps
2. Авторизуйтеся з вашим номером телефону
3. Створіть нову додаток (якщо ще не створили)
4. Скопіюйте **API ID** та **API Hash**

#### 1.2 ID вашого каналу
1.創ріть приватний канал або використовуйте існуючий
2. Від меню каналу виберіть "Інформація" → скопіюйте ID

**Альтернативно:** Напишіть у канал повідомлення, перейдіть на сайт `https://api.telegram.org/bot<YOUR_API_ID>/getUpdates`, знайдіть `chat` → `id` (негативне число для приватного каналу)

### Крок 2: Клонування та налаштування

```bash
# Клонуємо репозиторій
git clone https://github.com/smm-creator/agentga-analitcs.git
cd agentga-analitcs

# Копіюємо приклад .env
cp .env.example .env

# Редагуємо .env своїми значеннями
# Linux/Mac:
nano .env

# Windows: відкрийте .env у текстовому редакторі (Notepad, VS Code тощо)
```

### Крок 3: Запуск

**Linux/Mac:**
```bash
chmod +x run.sh
./run.sh
```

**Windows:**
```bash
run.bat
```

**Вручну:**
```bash
# Створюємо віртуальне середовище
python3 -m venv venv

# Активуємо (Linux/Mac):
source venv/bin/activate

# Активуємо (Windows):
venv\Scripts\activate

# Встановлюємо залежності
pip install -r requirements.txt

# Запускаємо клієнту
python3 user_client.py
```

## 📝 Налаштування .env файлу

```env
# Pyrogram Session Name - імя сесії (буде створена автоматично)
SESSION_NAME=my_session

# Telegram Channel ID - ID вашого каналу для публікації
# Формат: -100123456789 для приватних каналів або @channel_username
CHANNEL_ID=-100123456789

# API ID - отримайте на https://my.telegram.org/apps
API_ID=123456789

# API Hash - отримайте на https://my.telegram.org/apps
API_HASH=0123456789abcdefghijklmnopqrstuv
```

## 🔑 Налаштування ключових слів та посилань

Змініть словник `KEYWORD_LINKS` у файлі `user_client.py` (рядки ~40-49):

```python
KEYWORD_LINKS = {
    "штани": "https://prof1group.ua/pants",
    "взуття": "https://prof1group.ua/shoes",
    "рубашка": "https://prof1group.ua/shirts",
    "шапка": "https://prof1group.ua/hats",
    "жакет": "https://prof1group.ua/jackets",
    # Додавайте свої ключові слова тут
}
```

**Як додати нове слово:**
```python
"ваше_слово": "https://посилання-на-товар.ua",
```

## 📖 Як користуватися

### Текстові постинги
1. Надішліть приватне повідомлення клієнту (пошук клієнту по імені сесії)
2. Напишіть текст
3. Наприклад: "Купіть якісні штани та взуття в нас!"
4. Клієнт автоматично замініть слова на посилання та опублікує в канал

**Результат у каналі:**
```
Купіть якісні штани та взуття в нас!
```
(Слова будуть клікабельними посиланнями)

### Публікація фото
1. Надішліть фото приватно клієнту
2. Додайте опис у полі "Підпис"
3. Клієнт обробить опис та опублікує фото з форматованим текстом

### Команди
- `/start` - вітання та інформація
- `/help` - детальна інструкція
- `/keywords` - показати список всіх ключових слів

## 🔐 Перша авторизація

Піс запуску клієнту:
1. Вам буде запропоновано ввести номер телефону (в форматі +380...)
2. Вам прийде код підтвердження в Telegram
3. Введіть цей код
4. Сесія буде збережена в папці `sessions` (не видаляйте!)
5. При наступних запусках авторизація не потрібна

## 🐛 Розв'язування проблем

### "❌ Необхідно встановити API_ID, API_HASH та CHANNEL_ID"
- Перевірте, що файл `.env` існує в робочій папці
- Перевірте, що всі три значення заповнені коректно
- Перезапустіть клієнту

### Клієнт не відповідає на повідомлення
- Перевірте, що клієнт працює (в терміналі мають бути логи)
- Перевірте наявність інтернету
- Переконайтеся, що ви надсилаєте повідомлення приватно (не в групу/канал)
- Перезапустіть клієнту

### "❌ Помилка при публікації"
- Перевірте, що ви є адміністратором каналу
- Перевірте, що CHANNEL_ID вказаний коректно
- Для приватного каналу ID повинен починатися з `-100`
- Переконайтеся, що ви видалили попередній файл сесії при зміні акаунту

### Перша авторизація не працює
- Переконайтеся, що використовуєте вірний номер телефону (з кодом країни: +380...)
- Перевірте, що ввели правильний код підтвердження
- Видаліть файли сесії в папці `sessions` і спробуйте знову

## 📚 Технічні деталі

### Використовувані технології
- **Pyrogram 1.4.16** - асинхронна бібліотека для роботи з Telegram User API
- **asyncio** - асинхронне програмування в Python
- **python-dotenv** - завантаження конфігурації з .env файлу
- **RE (regex)** - пошук та заміна ключових слів

### Як працює заміна ключових слів
1. Використовується регулярний вираз з прапорцем `re.IGNORECASE`
2. Пошук здійснюється на границях слова (`\b`)
3. Оригінальний регістр зберегуно через `match.group(1)`
4. HTML-спеціальні символи екрануються для безпеки
